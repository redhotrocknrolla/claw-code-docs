# Advanced Note

## Причина выбора модуля

Модуль `rust/crates/tools/src/lib.rs` является центральным компонентом системы инструментов Claw Code. Он предоставляет:

- Регистрацию и управление встроенными инструментами
- Выполнение инструментов с проверкой разрешений
- Нормализацию имен и поддержку псевдонимов
- Создание API-спецификаций

Архитектура пайплайна документирования была выбрана для обеспечения:

- **Четкого разделения ответственности:** каждый skill выполняет свою задачу
- **Валидации между этапами:** каждый шаг проверяет артефакты предыдущего
- **Трассируемости:** все решения фиксируются в журналах

## Архитектура skill-based пайплайна

```
Orchestrator
    ↓
rust-module-analyzer → analyzer-output.md
    ↓
rust-doc-writer → doc-draft.md
    ↓
rust-style-auditor → tools-lib-doc-advanced.md
```

### Роль analyzer

Роль analyzer критична для качества документации:

- **Извлечение фактов:** только проверяемые по коду данные
- **Отказ от догадок:** если информация неочевидна — пометка "требует уточнения"
- **Структурирование:** четкое разделение публичных/непубличных сущностей
- **Вторичная проверка:** имитация эксперта Rust Auditor для выявления ошибок

### Контроль стиля

Роль auditor обеспечивает:

- **Язык:** русский для основного текста, английский только для идентификаторов
- **Тон:** официальный, технический, без разговорных формулировок
- **Структура:** строгая структура с обязательными разделами
- **Терминология:** единообразное использование терминов (инструмент, реестр, разрешения)

## Проблемные места

Где модель могла ошибиться:

1. **Типы из внешних модулей:** без анализа runtime, plugins, api невозможно точно описать
2. **Сценарии использования:** примеры основаны на коде, но могут не отражать реальное использование
3. **Ограничения:** не все ограничения могут быть очевидны из кода

## Трассируемость

Все решения зафиксированы в:

- `docs/dialogs/orchestrator-session.md` — общий журнал
- `docs/dialogs/analyzer-session.md` — журнал анализа
- `docs/dialogs/writer-session.md` — журнал генерации
- `docs/dialogs/auditor-session.md` — журнал аудита
- `docs/notes/advanced-note.md` — архитектурное пояснение

Каждое решение сопровождается контекстом и причинами выбора.
